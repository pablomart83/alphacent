# AlphaCent End-to-End Audit — 2026-05-01

**Auditor:** Kiro
**Basis:** Read of current code on EC2, current DB state, live logs (errors.log, cycle_history.log, execution.log, data.log), current config files. Steering file and Session_Continuation.md used only for context; findings based on code/data.
**Market state at audit:** Friday, US markets pre-open, London open ~1 hour. Equity $475,886, 86 open positions, regime trending_up_weak.

---

## 1. Executive Summary

1. **Walk-forward validation has systemic look-ahead bias.** Across 173 active strategies, average test Sharpe is 2.70 vs train 0.79 (test > train by +1.92 on average; max test Sharpe 7.24). Real WF should have test ≤ train on average. The "test-dominant" and "excellent OOS" bypass paths in `strategy_proposer.py` let strategies through with train ≈ 0 / test ≥ 1.0, which is pure regime luck, not edge. **Every activation decision made through these bypass paths is suspect.** Recommend tightening to strict `train AND test ≥ min_sharpe` and re-backtesting the active book.

2. **DST fix is incomplete — pipeline crashed again at 01:47 UTC today for 166 symbols.** Post-hoc `tz_convert` in `monitoring_service.py:1115` runs AFTER `yf.download()` returns, but `AmbiguousTimeError` fires INSIDE yfinance's own timestamp parsing before the return. Half the universe missed overnight 1h/1d refresh. The fix claimed "shipped" in Session_Continuation is architecturally wrong.

3. **Entry order fail rate is 82% last 7 days** (554 FAILED / 673 entries). Root cause: signals are marked FAILED in the `orders` table when the "Market closed — signal will re-fire" deferred path is hit, then re-fired each cycle creating more FAILED rows. Not a real failure — but it poisons every fill-rate and execution metric. Slippage column is populated 0/1204 in 14 days — we have zero visibility into actual execution quality.

4. **Symbol concentration cap (3%) not enforced.** NVDA and AMZN each at 7.43% of equity. CAT 3.72%, ROST 3.72%, AUDUSD 3.33%. The cap exists in config but doesn't reduce sizing for cumulative concurrent positions from different strategies on the same symbol.

5. **Multiple dead-code and config-vs-code divergence points.** `atr_sl_multiplier: 2.5` in config is never read (hardcoded `1.5` in `order_executor.py:241`). `fast_period`/`slow_period` substitution in `strategy_proposer.py:3059` collapses Triple-EMA templates into `EMA(10) > EMA(10)` always-false clauses — 0-trade templates. FXI, SPY data fetches fail with "possibly delisted" — Yahoo ticker mapping is wrong for multiple symbols.
